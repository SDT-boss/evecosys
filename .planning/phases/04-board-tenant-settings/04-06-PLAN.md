<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Storage API → tenant-assets bucket | Authenticated users can invoke DELETE/UPDATE on storage.objects — RLS must prevent cross-tenant operations |
| RSC session boundary | Layout auth guard and child RSC call createClient() independently — a session expiry window exists between the two |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-CR03-01 | Tampering | storage.objects DELETE on tenant-assets | mitigate | tenant_owner_delete policy: `split_part(name, '/', 2) = (SELECT id::text FROM public.tenants WHERE owner_id = auth.uid())` — ensures only the tenant owner's auth.uid() matches; cannot be spoofed without owning the tenants row |
| T-04-CR03-02 | Tampering | storage.objects UPDATE on tenant-assets | mitigate | tenant_owner_update policy: same predicate as CR03-01 — overwrites scoped to own tenant path |
| T-04-CR04-01 | Denial of Service | branding/page.tsx — user!.id panic on null | mitigate | `if (!user) redirect('/login')` before any field access — graceful redirect instead of 500 TypeError |
| T-04-CR04-02 | Denial of Service | toggles/page.tsx — user!.id panic on null | mitigate | Same redirect guard pattern |
| T-04-CR04-03 | Denial of Service | users/page.tsx — user!.id and tenant!.id panic | mitigate | Two guards: user null → redirect /login; tenant null → redirect /board/settings |
| T-04-CR-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
