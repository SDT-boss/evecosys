<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/board/settings/byodb | Untrusted BYODB credentials — must not be logged |
| BYODB API → BYODBRegistrationService | Service accesses external database — credentials stored in Vault only |
| Tenant state machine | Route must honour valid transitions only; cannot skip states |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Information Disclosure | BYODB credentials in error messages | mitigate | Error handler uses only `err.message` — BYODBRegistrationService is already designed to exclude credentials from error messages; API route does NOT log `req.body` or extracted credential fields |
| T-04-04-02 | Tampering | Tenant state machine bypass | mitigate | Route uses `transition()` from `lib/tenant/stateMachine.ts` which enforces TRANSITIONS table; if `Registered → Provisioning` is somehow invalid it throws `InvalidStateTransitionError` rather than silently proceeding |
| T-04-04-03 | Elevation of Privilege | BYODB route — cross-tenant registration | mitigate | Tenant fetched via `owner_id = auth.uid()` — board member can only register their own tenant; service client used for DB writes but within the scoped tenant.id |
| T-04-04-04 | Tampering | Credentials stored in tenants table | accept | Credentials are NOT stored in tenants table — SupabaseVault stores them in Vault (encrypted at rest); tenants table only stores state transition |
| T-04-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
