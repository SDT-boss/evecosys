## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|

| `var(--ds-space-md)` | 16px | Card internal padding, FormField stack gap, Alert padding, section content padding |
| `var(--ds-space-lg)` | 24px | Section gap between form groups, page content vertical rhythm |

| `var(--ds-space-3xl)` | 64px | Not used in this phase |

- Touch targets for Remove buttons (TableCell actions): minimum 40px height — use Button size="default" (h-10 = 40px) to satisfy this automatically.
- TabsList pill container: h-10 (40px) with p-1 internal — this is the Tabs component default; do not alter.

## Page-by-Page Interaction Contract

### Settings Layout (board/settings/layout.tsx)

- Replaces current passthrough layout.
- Retains existing auth + board role + tenant guard (unchanged).
- Adds a page-title area ("Tenant Settings", font-size-xl semibold ink) and the TabsList bar immediately below it.
- TabsList sits in a `<nav aria-label="Settings navigation">` wrapper.
- Four TabsTriggers rendered as Next.js `<Link asChild>`: Branding | Users | BYODB | Feature Toggles.
- Active trigger state derives from `usePathname()` matching the nested route prefix.
- `{children}` renders below the tab bar with `var(--ds-space-lg)` top gap.

### Branding Tab (/board/settings/branding)

**Server component** fetches tenant `{ display_name, logo_url, primary_color }` and passes to a Client component form.

Form sections (each wrapped in a Card):

1. **Tenant name**
   - Label: "Display name"
   - Input (text, type="text", maxLength={80}): pre-filled with current display_name
   - Helper text: "This name is shown to your users in the dashboard header."

2. **Logo**
   - Label: "Logo"
   - Visual treatment: simple `<input type="file" accept="image/png,image/jpeg,image/svg+xml">` wrapped in a custom styled drop zone div. The drop zone is a dashed-border rectangle (border: 1.5px dashed `var(--ds-color-neutral-grey-20)`, radius: `var(--ds-radius-md)`, padding: `var(--ds-space-lg)`). Inside: Upload icon (lucide `Upload`, 24px, grey-40 colour) + helper text "PNG, JPEG, or SVG — max 2 MB" in font-size-sm grey-60. On file selection, show the selected filename in grey-60.
   - If `logo_url` exists: show a 48x48 preview `<img>` + "Remove logo" ghost Button alongside the upload zone.
   - Helper text: "Uploaded logos are stored securely and accessible to your tenant users."

3. **Primary colour**
   - Label: "Primary colour"
   - Input (text, type="text", fontFamily mono, placeholder="#008684", maxLength={7})
   - Client-side validation: must match `/^#[0-9A-Fa-f]{6}$/`. On invalid: aria-invalid="true", error shadow applied, inline error message "Enter a valid 6-digit hex colour, e.g. #1A2B3C."
   - Normalise to uppercase on blur.
   - Colour swatch preview: 24x24 div with border-radius `var(--ds-radius-sm)`, border `1px solid var(--ds-color-neutral-grey-20)`, background set to the entered hex value (only when valid). Renders inline to the right of the Input.
   - Helper text: "Used to tint interactive elements for your tenant's branding. Only stored — not applied in v1 (preview deferred)."

**Save button:**
- Label: "Save branding"
- Variant: `secondary` (Jade, white label)
- Positioned at bottom of the form, left-aligned.
- During submission: Spinner replaces label text, button is disabled.
- On success: Alert (success variant) above the button: Title "Branding saved", Description "Your tenant's branding settings have been updated."
- On failure: Alert (destructive variant) above the button: Title "Save failed", Description: the error message from ActionResult.

### Users Tab (/board/settings/users)

**Server component** fetches `users WHERE tenant_id = {tenantId}` — returns `{ id, name, email, role }[]`. Checks if `feature_flags.auth_troubleshooting` is true.

**Invite section (Card at top):**
- Heading: "Invite a member" (font-size-lg semibold)
- FormField: Email — Label "Email address", Input type="email", placeholder "colleague@example.com"
- FormField: Role — Label "Role", Select with two options:
  - "Manager" (value: "manager")
  - "Driver" (value: "driver")
  - Default selected: "Manager"
- CTA: Button variant="default" (Volt Green) label "Send invitation"
- On submit: Spinner in button, disabled. On success: Alert success "Invitation sent — {email} will receive an email shortly." On failure: Alert destructive "Invitation failed — {error}."

**Member list (Card below invite section):**
- Section heading: "Team members" (font-size-lg semibold)
- Table columns: Name | Email | Role | Actions
- Name cell: font-weight medium, ink colour.
- Email cell: font-size-sm, grey-60.
- Role cell: Badge — Manager = `default` variant (Jade), Driver = `secondary` variant (grey).
- Actions cell: Contains "Remove" button (variant="destructive", size="sm"). If `auth_troubleshooting` flag is ON, also contains "Force password reset" button (variant="outline", size="sm", label="Reset password").
- Empty state (no users): EmptyState with icon=`Users` (lucide), title="No team members", description="Invite your first manager or driver using the form above."

**Remove user — AlertDialog:**
- Trigger: Remove button in table row.
- AlertDialogTitle: "Remove {user.name}?"
- AlertDialogDescription: "This will permanently remove {user.name} ({user.email}) from your tenant and revoke their access. This cannot be undone."
- AlertDialogAction: Button destructive "Remove member"
- AlertDialogCancel: Button outline "Cancel"
- On confirm: Spinner, disabled. On success: user row removed, Alert success "Member removed — {name} no longer has access to this tenant." On failure: AlertDialog closes, Alert destructive shown above the table.

**Force password reset (auth_troubleshooting = ON only):**
- No separate confirmation dialog — tooltip on hover suffices: "Send a password reset email to this user."
- On click: Button shows Spinner, then inline toast-equivalent: the row's Actions cell briefly shows a green checkmark icon with accessible text "Reset email sent" for 3 seconds, then reverts.
- On failure: Alert destructive above the table: "Password reset failed — {error}."

### BYODB Tab (/board/settings/byodb)

**Server component** fetches current tenant BYODB status. If no connection registered, page shows the registration form. If a connection exists, shows the connection summary + a "Re-register" option.

**No connection state — registration form (Card):**
- Heading: "Register database connection" (font-size-lg semibold)
- Mode toggle: Two-option RadioGroup or segmented control (use RadioGroup from design system): "Structured fields" (default) | "Connection string"
  - RadioGroup items render as a horizontal pair of bordered radio cards. Selected card: Jade left-border. Unselected: grey-20 border.

**Structured fields mode:**
- FormField: Engine — Select options "PostgreSQL" (value: postgres) | "MySQL" (value: mysql)
- FormField: Host — Input type="text", placeholder "db.example.com"
- FormField: Port — Input type="number", placeholder defaults from engine (5432 / 3306), min=1, max=65535
- FormField: Database — Input type="text", placeholder "mydb"
- FormField: Username — Input type="text", placeholder "dbuser"
- FormField: Password — Input type="password", autocomplete="new-password"
- Checkbox: "Use SSL" — Label "Require SSL connection" (default unchecked)
- Layout: Two-column grid (Host / Port side by side), full-width fields otherwise.

**Connection string mode:**
- FormField: Engine — Select (same as above)
- FormField: Connection string — Input type="text", fontFamily mono, placeholder "postgresql://user:pass@host:5432/dbname"
- Helper text: "Paste a full connection URI. Credentials are validated before saving."

**Register button:**
- Label: "Register connection"
- Variant: `secondary` (Jade)
- On submit: Spinner + disabled. Calls `/api/board/settings/byodb` which runs `registrationService.register()`.
- On success: Alert success Title "Connection registered", Description "Your database connection was verified and saved successfully."
- On failure: Alert destructive Title "Registration failed", Description: the specific error from the registration service (probe failure message, vault error, etc.)

**Connection registered state:**
- Shows a summary Card with: Engine badge (Badge outline), host (partially masked for security: show first segment only, e.g. "db.exa••••"), registration date in grey-60, status Badge "Active" (default/Jade variant).
- "Re-register connection" — Button variant="outline" label "Update connection". Clicking replaces the summary with the registration form pre-cleared.

**Empty state (initial):** No EmptyState component needed — the registration form is the primary content.

### Feature Toggles Tab (/board/settings/toggles)

**Server component** fetches `tenants.feature_flags` JSONB for the current tenant. Default: all 8 flags true.

**Layout:** Single Card containing two groups of Switches.

**Group 1 — Feature availability (7 flags):**
Heading: "Platform features" (font-size-lg semibold, margin-bottom var(--ds-space-md))

Each flag row:
- Layout: `flex items-center justify-between` row, padding `var(--ds-space-md)` top/bottom, border-bottom `1px solid var(--ds-color-neutral-grey-10)` (except last row).
- Left: flag label (font-size-sm medium ink) + description (font-size-sm grey-60, below label)
- Right: Switch

| Flag key | Label | Description |
|----------|-------|-------------|
| `fleet` | Fleet management | Vehicle tracking, assignments, and fleet health monitoring |
| `carbon` | Carbon reporting | CO₂ emissions tracking and environmental impact reports |
| `trips` | Trip management | Trip logs, route history, and driver trip assignment |
| `driver_behaviour_score` | Driver behaviour scoring | Safety and efficiency scores based on trip telemetry |
| `alerts` | Alerts | Real-time notifications for fleet events and anomalies |
| `charging_stations` | Charging stations | Charging point management and session monitoring |
| `member_invitations` | Member invitations | Allow board members to invite new managers and drivers |

**Group 2 — Administrative tools (1 flag):**
Heading: "Administrative tools" (font-size-lg semibold, margin-top `var(--ds-space-xl)`, margin-bottom `var(--ds-space-md)`)
- This group is visually separated to distinguish it from feature-availability flags.

| Flag key | Label | Description |
|----------|-------|-------------|
| `auth_troubleshooting` | Auth troubleshooting | Enables force password reset actions on the Users tab |

**Save behaviour:**
- No auto-save. A single "Save changes" button (variant="secondary", Jade) sits at the bottom of the Card.
- On save: PATCH `/api/board/settings/toggles` with full updated flags object. Spinner during submission.
- On success: Alert success Title "Feature flags saved", Description "Your feature settings have been updated."
- On failure: Alert destructive Title "Save failed", Description error message from ActionResult.

## Interaction States

### Button loading state (all Save / Submit actions)

When a form is submitting:
1. Button is set to `disabled`.
2. Button label text is replaced by `<Spinner size="sm" />` + visually hidden text (aria-label preserved).
3. Alert (if previously shown) remains visible until the new response arrives.
4. On response: re-enable button, show new Alert above button.

### Switch toggle latency

- Switch does not auto-save. Toggling a Switch updates local React state only.
- Visual change is immediate (optimistic local update).
- Actual persistence happens on "Save changes" button press.
- If the user navigates away without saving: no confirmation prompt (v1 — deferred).

### Hex colour validation

- Validated on blur and on submit.
- Pattern: `/^#[0-9A-Fa-f]{6}$/` (exactly 6 hex digits, # prefix required).
- Normalise to uppercase on blur (e.g. `#1a2b3c` → `#1A2B3C`).
- Empty field is valid (means no primary colour preference set — stores NULL).
- Swatch preview only renders when the current value is a valid hex string.

### BYODB mode switch

- Switching between "Structured fields" and "Connection string" modes clears all field values (no cross-population in v1).
- Mode choice is local state — not persisted until Register is clicked.

## Accessibility Contract

- All interactive controls have associated `<Label>` via `htmlFor`/`id`.
- AlertDialog (Remove user) uses `role="alertdialog"` provided by Radix — no additional role needed.
- Alert components render with `role="alert"` (provided by design system Alert).
- Switch components pair with `<Label htmlFor={...}>` — per design system usage pattern.
- Tab navigation bar: `<nav aria-label="Settings navigation">` wrapper.
- Inline form errors: set `aria-invalid="true"` on the Input + `aria-describedby` pointing to the error message element.
- Force password reset button: `title="Send a password reset email to {name}"` + `aria-label`.
- Colour swatch in Branding tab: `aria-hidden="true"` (decorative).
- Minimum touch target: 40px height — all Buttons use default size (h-10) or sm (h-8 — acceptable for table action rows).
- Focus rings: `var(--ds-color-brand-primary)` (Jade) on all interactive controls — already enforced by design system components.
