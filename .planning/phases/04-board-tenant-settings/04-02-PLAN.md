  const { data: { user } } = await supabase.auth.getUser()
  if (!user) redirect('/login')

  const { data: tenant } = await supabase
    .from('tenants').select('id').eq('owner_id', user.id).single()
  if (!tenant) redirect('/login')

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/board/settings/branding | Untrusted multipart body with arbitrary file content |
| board member → tenants row | Must only update their own tenant (owner_id guard) |
| Storage upload | Arbitrary file content must not execute or be served as script |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | POST /api/board/settings/branding — file upload | mitigate | Content-type validated against allowlist `['image/png','image/jpeg','image/svg+xml']` server-side; size capped at 2 MB; client-side `accept` attribute provides first-layer UX hint |
| T-04-02-02 | Elevation of Privilege | POST /api/board/settings/branding — tenant scope | mitigate | Route re-verifies board role via `supabase.auth.getUser()` (not session) + fetches tenant via `owner_id = auth.uid()` — cannot modify another tenant's row |
| T-04-02-03 | Tampering | primary_color — hex injection | mitigate | Server validates `/^#[0-9a-fA-F]{6}$/` before writing; empty string is accepted (NULL stored); no SQL interpolation — Supabase client uses parameterised queries |
| T-04-02-04 | Information Disclosure | SettingsTabNav — auth bypass | accept | Tab nav is purely navigational; auth guard in `board/settings/layout.tsx` blocks unauthenticated RSC render regardless of tab URL; no sensitive data in the nav component itself |
| T-04-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
