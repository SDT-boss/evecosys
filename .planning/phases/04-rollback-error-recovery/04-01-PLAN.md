<objective>
Harden the BYODB provisioning rollback path so that a post-store failure (1) explicitly resets tenant state to `Registered` in memory, (2) deletes the orphaned Vault secret, and (3) surfaces a `ProvisioningRollbackError` wrapping both the original and the rollback error when Vault cleanup itself fails. Migrate and expand rollback coverage into a dedicated `rollback.test.ts`.

Purpose: Satisfies ROLLBACK-01 (failure triggers rollback), ROLLBACK-02 (state resets to Registered, partial state wiped), ROLLBACK-03 (no orphaned credentials in Vault). This is hardening — the rollback plumbing already exists in `registrationService.ts` lines 60-70; this plan makes it explicit, testable, and dual-failure-safe.
Output: `ProvisioningRollbackError` class, hardened `register()` catch block, new `rollback.test.ts`, and removal of the legacy inline rollback test block from `registrationService.test.ts`.
</objective>
