       - `async probe(params: DbConnectionParams): Promise<ProbeResult>`:
         - For `params.engine === 'postgres'`: dynamically `const { Client } = await import('pg')`, construct `new Client({ host, port, database, user, password, ssl: params.ssl ? { rejectUnauthorized: false } : undefined, connectionTimeoutMillis: 5000 })`, `await client.connect()`, run `await client.query('SELECT 1')` for reachability and `await client.query('SELECT has_schema_privilege(current_user, $1, $2)', [params.database, 'CREATE'])` — wait, schema ownership check: run `await client.query("SELECT has_database_privilege(current_user, current_database(), 'CREATE') AS owns")` and set `ownsSchema` from the boolean result. Always `await client.end()` in a finally block.
         - For `params.engine === 'mysql'`: dynamically `const mysql = await import('mysql2/promise')`, `const conn = await mysql.createConnection({ host, port, database, user, password, connectTimeout: 5000, ssl: params.ssl ? {} : undefined })`, run `await conn.query('SELECT 1')` for reachability and `await conn.query('SHOW GRANTS FOR CURRENT_USER()')` then set `ownsSchema = true` if any grant row contains `CREATE` or `ALL PRIVILEGES`. Always `await conn.end()` in finally.
         - On any thrown error, return `{ reachable: false, ownsSchema: false, error: <error.message> }` — do NOT include params.password in the error string.
       - Use `await import(...)` (not top-level import) so the test suite, which never instantiates RealConnectivityProbe, does not need the driver loaded.

  <behavior>
    - Happy path: a structured/postgres input with a probe returning {reachable:true, ownsSchema:true} → vault.store is called once → tenant.state becomes 'Active' → result includes the returned secretId.
    - Connectivity failure: probe returns {reachable:false} (or {ownsSchema:false}) → throws ConnectivityError → vault.store is NEVER called → tenant.state stays 'Provisioning'.
    - Invalid input: normalizeCredential throws CredentialValidationError → probe is NEVER called.
    - Rollback: probe succeeds, vault.store succeeds (returns secretId), but the subsequent transition (or a thrown post-store step) fails → vault.delete is called exactly once with that secretId → the original error propagates.
    - register requires the tenant to be in 'Provisioning' state; if not, it throws before probing.
  </behavior>
  <action>
    Create `lib/tenant/registrationService.ts` exporting `export class BYODBRegistrationService`:
    - Imports: `normalizeCredential, type BYODBCredentialInput, CredentialValidationError` from `@/lib/tenant/credentials`; `type ConnectivityProbe, ConnectivityError` from `@/lib/tenant/probe`; `type VaultStore` from `@/lib/tenant/vault`; `transition` from `@/lib/tenant/stateMachine`; `type Tenant` from `@/lib/tenant/types`.
    - Constructor `(private readonly probe: ConnectivityProbe, private readonly vault: VaultStore)`.
    - `export interface RegistrationResult { tenant: Tenant; secretId: string }` (export at top level).
    - `async register(tenant: Tenant, input: BYODBCredentialInput): Promise<RegistrationResult>`:
      1. If `tenant.state !== 'Provisioning'` throw `new ConnectivityError(\`Tenant ${tenant.id} must be in Provisioning to register credentials, got ${tenant.state}\`)`.
      2. `const params = normalizeCredential(input)` — lets CredentialValidationError propagate before any probe/storage.
      3. `const result = await this.probe.probe(params)`. If `!result.reachable || !result.ownsSchema` throw `new ConnectivityError(\`BYODB connectivity probe failed for tenant ${tenant.id}: ${result.error ?? (!result.reachable ? 'unreachable' : 'no schema ownership')}\`)`. (vault.store is unreachable from this branch.)
      4. `const stored = await this.vault.store(\`byodb/${tenant.id}\`, JSON.stringify(params))`.
      5. Wrap the post-store work in try/catch for rollback:
         ```
         try {
           const nextState = transition(tenant.state, 'Active')
           return { tenant: { ...tenant, state: nextState, updated_at: new Date().toISOString() }, secretId: stored.secretId }
         } catch (err) {
           await this.vault.delete(stored.secretId)
           throw err
         }
         ```
      Never log `params` or `input` (they contain the password). The secret string passed to vault.store is the only place credentials are serialized.
  </action>
