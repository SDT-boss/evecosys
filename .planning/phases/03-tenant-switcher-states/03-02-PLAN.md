<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TenantList click → setActiveTenant Server Action | User-initiated action sends a tenantId (UUID from server-rendered tenant list) to the server; server validates via RLS before writing cookie |
| setActiveTenant → browser cookie | Server Action writes platform_active_tenant cookie; httpOnly: false by design (D-07) so client can read for optimistic UI |
| TenantContext → ActiveTenantIndicator | Context value (tenant name string) flows from server-rendered prop → TenantProvider → context consumer; display only, not a security gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-W1-01 | Spoofing | platform_active_tenant cookie value | accept | Cookie stores tenant UUID; RLS policies (Phase 1) scope all queries to the active tenant; an attacker setting an arbitrary UUID either gets no data (UUID not in tenants table) or their own tenant's data (already accessible). httpOnly: false is intentional (D-07). |
| T-03-W1-02 | Tampering | Double-submit race via rapid row clicks | mitigate | D-02 (per CONTEXT.md): pointer-events: none on all TableRows during isPending prevents a second click from firing a second transition. Only one transition runs at a time. |
| T-03-W1-03 | Information Disclosure | switchError displayed to admin | accept | Error string comes from our own server action (cookieStore.set error message), not from external input; it is displayed only to the platform admin who initiated the action |
| T-03-W1-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 3; slopcheck not required |
</threat_model>
