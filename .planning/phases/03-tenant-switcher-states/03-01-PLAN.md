<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Next.js middleware → RSC layout | `x-pathname` header is set by our own middleware and consumed by `platform/layout.tsx` — trusted internal boundary; no user input reaches this header |
| Browser → TenantContext | Context values are set from server-rendered prop (`initialName`) or from `setActiveTenantName` called after a verified server action result — not from raw user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-W0-01 | Spoofing | middleware.ts `x-pathname` header | accept | Header is set server-side by our middleware, not by the client; Next.js middleware runs before any user-controlled request data reaches the layout; low risk |
| T-03-W0-02 | Tampering | TenantContext `activeTenantName` state | accept | Context state is derived from server-rendered prop (layout RSC reads cookie) and optimistic updates from `setActiveTenant` ActionResult; cookie is validated server-side; context value is display-only (tenant name), not a security gate |
| T-03-W0-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 3 — all dependencies already in package.json; Package Legitimacy Audit in RESEARCH.md confirms zero new installs |
</threat_model>
