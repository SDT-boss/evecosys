## Color

All surface values map to CSS variables declared in `app/globals.css` (legacy chrome variables) and `design-system/tokens/variables.css` (ds tokens). The PlatformShell is a dark-topbar / dark-nav chrome identical to DashboardShell.

### 60/30/10 Split

| Role | CSS Variable | Light value | Dark value | Usage |
|------|-------------|-------------|-----------|-------|
| Dominant surface (60%) | `var(--bg)` | `#f2f5f2` | `#111412` | Page background behind tenant list |
| Secondary surface (30%) | `var(--topbar-bg)` + `var(--nav-bg)` | `#1a1a1a` / `#242424` | `#0d0f0d` / `#141614` | PlatformShell topbar + horizontal nav |
| Accent (10%) | `var(--ds-color-brand-primary)` — Cyber Jade `#008684` | — | — | Active nav tab underline + text only |

**Accent reserved for:**
1. Active horizontal nav tab: bottom-border `2.5px solid var(--ds-color-brand-primary)` + text colour
2. `ActiveTenantIndicator` active state: Jade-tinted icon (BuildingOffice) in indicator slot
3. Focus rings on interactive elements (row click targets, nav tabs)

**Badge color mapping — status semantic:**

| Display label | Badge variant | Background token | Text |
|---------------|--------------|-----------------|------|
| Active | `default` | `var(--ds-color-brand-primary-strong)` (#007069) | white |
| Pending | `secondary` | `var(--ds-color-neutral-grey-10)` (#E6EAEA) | ink |
| Suspended | `destructive` | `var(--ds-color-status-error)` (#EF4444) | white |

> Source: `Badge/index.tsx` variant definitions. `default` = Jade (Active/confirmed), `secondary` = neutral grey (Pending/inactive), `destructive` = error red (Suspended/critical).

**Status mapping applied (D-05 from CONTEXT.md + Claude's discretion for Decommissioned):**

| `tenants.state` DB value | Display label | Badge variant |
|--------------------------|--------------|--------------|
| `Registered` | Pending | `secondary` |
| `Provisioning` | Pending | `secondary` |
| `Active` | Active | `default` |
| `Suspended` | Suspended | `destructive` |
| `Decommissioned` | **Filtered out** — not shown in list | — |

**Decommissioned decision (Claude's discretion):** Filter `Decommissioned` tenants from the list entirely. Rationale: platform admins taking active action on a decommissioned tenant is out of scope for Phase 2 (PADM-06 is a v2 requirement). Showing them creates confusion without any actionable path. The DB query adds `WHERE state != 'Decommissioned'` to keep the list operationally meaningful.

**No destructive actions in Phase 2.** The row-click action only sets a cookie — it does not delete, suspend, or modify tenant data. No destructive color use needed.

> Source: `Badge/index.tsx`, `tokens.json`, CONTEXT.md D-05, REQUIREMENTS.md PADM-06 (v2)

## Interaction Contract

### PlatformShell — Topbar + Nav

- Topbar: `h-[62px]`, `sticky top-0 z-30`, background `var(--topbar-bg)`, bottom border `1px solid var(--topbar-border)`
- Left slot: EVEcosys `<Logo>` + `1px` divider + label `"Platform Admin"` (12px, grey-40)
- Right slot: `<ActiveTenantIndicator>` + `<ThemeToggle>` + user avatar + logout button
- Nav strip: background `var(--nav-bg)`, bottom border `1px solid var(--topbar-border)`, `px-[var(--ds-space-xl)]` horizontal padding
- Active nav tab: `color: var(--ds-color-brand-primary)`, `border-bottom: 2.5px solid var(--ds-color-brand-primary)`, `margin-bottom: -1px`
- Inactive nav tab: `color: var(--ds-color-neutral-grey-60)`, `border-bottom: 2.5px solid transparent`
- Nav tab transition: `transition-colors duration-[var(--ds-motion-duration-fast)]` (100ms); weight is `--ds-font-weight-semibold` (600) for all tab states

### ActiveTenantIndicator — Header Slot

**No active tenant (placeholder state):**
- Text: `"No workspace selected"` — 12px, weight 600 (semibold), color `var(--ds-color-neutral-grey-40)` (#8A9290)
- Preceded by a `BuildingOffice2` icon (Lucide, 14px) at `var(--ds-color-neutral-grey-40)` opacity
- No background pill — inline text only, sits left of ThemeToggle

**Active tenant state:**
- Text: tenant display name from `tenants.name` — 12px, weight 600 (semibold), color `var(--ds-color-brand-primary)` (Jade)
- Preceded by `BuildingOffice2` icon (Lucide, 14px) at `var(--ds-color-brand-primary)` color
- Transition on update: `transition: color var(--ds-motion-duration-base) var(--ds-motion-easing-standard)` (200ms ease) — color fades from grey-40 to Jade when tenant becomes active
- No animation on the icon; no scale or translate — color-only transition keeps it subtle

**Animation decision (Claude's discretion):** Color-only transition at 200ms (`--ds-motion-duration-base`) on both icon and text. No layout shift, no scale/bounce. Rationale: the indicator lives in a sticky header; motion must not be distracting. Color transition is sufficient to communicate state change without drawing focus away from the table below.

### TenantList — Row Click

- Entire `<TableRow>` is the click target: `cursor-pointer`, `role="button"`, `tabIndex={0}`
- Keyboard: `Enter` and `Space` trigger the same Server Action as click
- Row height: minimum 56px (`h-14`) — inherits Table's standard row height (Table component convention)
- On click: Server Action sets `platform_active_tenant` cookie; Next.js `revalidatePath('/platform')` re-renders layout; `ActiveTenantIndicator` updates via server re-render (no client state needed in Phase 2)
- Active row (currently selected tenant): `data-[state=selected]` applied → `bg-[var(--ds-color-neutral-grey-10)]` (Table's built-in selected style)
- Hover: `hover:bg-[var(--ds-color-neutral-grey-05)]` (Table's built-in hover style)
- No confirmation dialog for row-click — it is a non-destructive context switch

### Loading State

- Show 5 `<Skeleton>` rows at 56px height inside `<TableBody>` while server is fetching
- Each skeleton row has 2 cells: `<Skeleton className="h-4 w-48" />` (name) + `<Skeleton className="h-5 w-20 rounded-full" />` (badge)

## Decisions Made (Claude's Discretion)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Decommissioned tenant visibility | Filtered out — excluded from DB query | No actionable path in Phase 2; PADM-06 (v2) |
| ActiveTenantIndicator placeholder copy | `"No workspace selected"` | Neutral, non-alarming; signals absence without implying error |
| Status badge colour for Pending | `secondary` variant (grey-10 bg, ink text) | Pending is neutral/informational — not a success or failure state |
| Status badge colour for Suspended | `destructive` variant (error red) | Suspended is an operationally critical state; red signals action needed |
| ActiveTenantIndicator transition | 200ms color-only on icon + text | Subtle enough for sticky header; no layout shift |
| Row click selected state | `data-[state=selected]` on TableRow | Uses Table's built-in selected style; no custom CSS needed |
