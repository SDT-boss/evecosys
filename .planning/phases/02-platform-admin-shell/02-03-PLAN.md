<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client TenantList → Server Action | tenantId crosses client→server; it is a DB-verified UUID from a rendered row, not free-form input |
| Server Action → cookie store | setActiveTenant writes platform_active_tenant (httpOnly:false per D-07) |
| page.tsx (server) → tenants table | RLS policy tenants_select_platform_admin gates the SELECT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-06 | Tampering | `setActiveTenant` cookie write | mitigate | Cookie write happens ONLY inside the `'use server'` action (never a Server Component) — the only place Next.js permits `cookies().set()` (02-RESEARCH Pitfall 2). Value is a UUID; httpOnly:false is the locked D-07 decision (client must read it for Phase 3 optimistic UI). The cookie alone grants no data access; RLS + JWT role gate all reads. (ASVS V3) |
| T-2-07 | Information Disclosure / Cross-tenant leakage | `page.tsx` tenants query | mitigate | Query uses plain `createClient()` so the `tenants_select_platform_admin` RLS policy applies — only platform admins receive rows, and the policy scopes what is returned. The service-role client (which bypasses RLS) is deliberately NOT used (acceptance criterion enforces this). Decommissioned rows excluded via `.neq('state','Decommissioned')` for least-information display. (ASVS V4 Access Control) |
| T-2-08 | Input Validation | `setActiveTenant(tenantId)` | accept | tenantId originates from a DB-rendered table row (already a valid UUID); no external user input. Worst case of a forged cookie UUID is `activeTenantName` resolving to null (placeholder). Low risk, no data exposure. (ASVS V5) |
| T-2-09 | Spoofing | row-click reaching the action without auth | mitigate | The /platform layout guard (Plan 02, T-2-03) gates the entire route; an unauthenticated/wrong-role user is redirected at the layout before TenantList or the action are reachable. (ASVS V4) |
| T-2-SC | Tampering | npm installs | accept | Zero new packages this phase (02-RESEARCH Package Legitimacy Audit). |
</threat_model>
