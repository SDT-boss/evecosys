<objective>
Deliver the visible Platform Admin shell. After this plan, a platform admin navigating to `/platform` sees the bespoke `PlatformShell` (dark topbar + horizontal "Tenants" nav tab) with the `ActiveTenantIndicator` in the header. With no active tenant cookie, the indicator shows "No workspace selected"; if a `platform_active_tenant` cookie is already present, the layout reads it and the header shows that tenant's name.

This is the second slice of the vertical: the shell chrome + persistent header indicator (PADM-03). The row-click Server Action that SETS the cookie is delivered in Plan 03; here the layout READS the cookie and renders the chrome. The shell can be verified with a manually-set cookie before Plan 03 wires the click.

Purpose: Render the persistent shell + active-tenant header indicator (PADM-03).
Output: 2 new components + extended platform layout.
</objective>

## Trust Boundaries

| Boundary | Description |

| layout (server) → PlatformShell (client) | activeTenantName prop passed server→client; no secrets exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-03 | Elevation of Privilege | `app/(dashboard)/platform/layout.tsx` guard | mitigate | Role guard (`profile.role !== 'platform_admin'` → redirect `/login`) runs on every request BEFORE the cookie read. Layout is dynamically rendered (forced by `cookies()`), so the auth check cannot be statically cached/bypassed. (ASVS V4 Access Control) |
| T-2-04 | Spoofing / Information Disclosure | `platform_active_tenant` cookie read | mitigate | Cookie value is a UUID only — no secrets/PII. An arbitrary/forged UUID that matches no row resolves `activeTenantName` to null (indicator falls back to placeholder); the cookie grants NO data access on its own — RLS + the user's JWT role gate all tenant data (ASVS V3 Session Management). httpOnly:false is set in Plan 03 per locked D-07; this plan only reads, never sets, the cookie. |
| T-2-05 | Tampering | layout cookie write | mitigate | Layout is a Server Component and only calls `cookieStore.get()`; it never calls `.set()` (which would throw and is reserved for the Server Action). Enforced by acceptance criterion grep for absence of `cookieStore.set(`. |
| T-2-SC | Tampering | npm installs | accept | Zero new packages this phase (02-RESEARCH Package Legitimacy Audit). |
